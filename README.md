## ğŸ§ About the Project

The **Inventory Management System** is designed to streamline operations for businesses managing multiple companies, employees, and stock inventories. It replaces manual spreadsheet tracking with a secure, centralized web application.

Built with a **Microservices-ready** mindset using a Monolithic architecture for simplicity, it leverages the power of **Spring Boot** for a high-performance REST API and **Angular** for a dynamic, responsive client interface.

## ğŸš€ Key Features

*   **ğŸ” JWT Authentication & Authorization**
    *   Secure stateless session management using JSON Web Tokens.
    *   **Role-Based Access Control (RBAC):** Distinct permissions for `ADMIN` and `USER` roles.
    *   Custom Route Guards in Angular to protect client-side navigation.

*   **ğŸ“¦ Smart Inventory Tracking**
    *   **Real-time Stock Calculations:** Automated updates for `Stock Value` and `Reorder Status`.
    *   **Low Stock Alerts:** Visual indicators when stock falls below `Minimum` or `Buffer` levels.
    *   **Transaction History:** Track `Receipts` (incoming) and `Issues` (outgoing).

*   **ğŸ¢ Multi-Tenancy Support (Logical)**
    *   Manage multiple **Companies** within a single system.
    *   Associate **Employees** and **Inventory** items to specific companies.

*   **ğŸ“Š Interactive Dashboards**
    *   **Admin Dashboard:** High-level overview of system metrics.
    *   **User Dashboard:** Operational views for day-to-day tasks.

## âš¡ Getting Started

Follow these steps to set up the project locally.

### 1. Cloning the Repository

Open your terminal and run the following command to download the codebase:

```bash
git clone https://github.com/your-username/inventory-management-system.git
cd inventory-management-system
```

*(Note: Replace the URL with the actual repository URL if hosted remotely)*

### 2. Database Setup

1.  Open **pgAdmin** or your preferred SQL tool (DBeaver, terminal).
2.  Create a new database named `finaldb`:
    ```sql
    CREATE DATABASE finaldb;
    ```
    *The tables will be automatically generated by Hibernate when the backend starts.*

### 3. Backend Setup (Spring Boot)

1.  Navigate to the backend folder:
    ```bash
    cd backend
    ```
2.  **Configure Database Credentials:**
    Open `src/main/resources/application.properties` and update the username/password if yours differs from the default:
    ```properties
    spring.datasource.username=postgres
    spring.datasource.password=your_password_here
    ```
3.  **Run the Application:**
    *   **Windows:**
        ```powershell
        .\mvnw spring-boot:run
        ```
    *   **Mac/Linux:**
        ```bash
        ./mvnw spring-boot:run
        ```
4.  The server will start on `http://localhost:8080`.

### 4. Frontend Setup (Angular)

1.  Open a new terminal window and navigate to the frontend folder:
    ```bash
    cd frontend
    ```
2.  **Install Dependencies:**
    ```bash
    npm install
    ```
3.  **Start the Development Server:**
    ```bash
    ng serve
    ```
4.  Open your browser and visit: `http://localhost:4200`

## ğŸ”§ Troubleshooting

*   **Database Connection Refused:**
    *   Ensure PostgreSQL service is running.
    *   Verify credentials in `application.properties`.
    *   Check if the port `5432` is correct.
*   **CORS Error in Browser:**
    *   The backend is configured to accept requests from `http://localhost:4200`. If you run Angular on a different port, update `SecurityConfig.java` in the backend.
*   **Port 8080 is In Use:**
    *   Change the server port in `application.properties`:
        ```properties
        server.port=9090
        ```
    *   *Remember to update `environment.ts` in the frontend if you change this.*
